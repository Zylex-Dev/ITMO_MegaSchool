# Multi-Agent Interview Coach - Руководство по установке

Этот гайд поможет вам настроить и запустить **AI Interview Coach** — систему, имитирующую техническое собеседование с использованием нескольких автономных агентов (Интервьюер, Наблюдатель, Стратег) и системой логирования.

## Требования
- **Python 3.10** или выше.
- **OpenAI API Key** (Для работы требуется доступ к GPT-4o или GPT-4-turbo).

## Инструкция по установке

### 1. Настройка проекта
Перейдите в директорию проекта:
```bash
cd c:/Other_Tools/ITMO_MegaSchool
```

### 2. Создание виртуального окружения (Рекомендуется)
Использование виртуального окружения поможет избежать конфликтов зависимостей.
```bash
# Windows
python -m venv venv
.\venv\Scripts\activate

# Mac/Linux
python3 -m venv venv
source venv/bin/activate
```

### 3. Установка зависимостей
Установите необходимые пакеты с помощью `pip`:
```bash
pip install -r requirements.txt
```

### 4. Настройка API ключей
1. Откройте файл `.env` в корневой директории.
2. Замените заглушку на ваш реальный OpenAI API Key.
```ini
OPENAI_API_KEY=sk-proj-ВАШ_РЕАЛЬНЫЙ_КЛЮЧ
# Опционально: LangSmith tracing
LANGCHAIN_TRACING_V2=true
LANGCHAIN_API_KEY=lsv2_ВАШ_КЛЮЧ_LANGCHAIN
```

## Как запустить

### Вариант А: Графический интерфейс (GUI) - Рекомендуется
Запускает современный веб-интерфейс с боковой панелью "Скрытые мысли" (Hidden Thoughts).
```bash
streamlit run src/app.py
```
- Откройте браузер по ссылке (обычно `http://localhost:8501`).
- Общайтесь с AI в главном окне чата.
- Раскройте боковую панель (Sidebar), чтобы видеть **анализ в реальном времени** (Техническая и Поведенческая оценка).
- Нажмите **"End Session & Generate Report"**, чтобы завершить интервью и получить подробный фидбек.

### Вариант Б: Консольный режим (CLI)
Для тестирования логики агентов в терминале без графического интерфейса:
```bash
python src/main.py
```
- Вводите ответы и нажимайте Enter.
- Внутренние "мысли" агентов будут выводиться прямо в консоль.
- Введите `exit` для завершения.

## Логи и отчеты
Каждая сессия автоматически сохраняется в файл `interview_log.json` в корне проекта.
- Файл содержит полную транскрипцию диалога, скрытые "мысли" агентов и финальный отчет.
- **Формат**: JSON (соответствует требованиям задания).

## Возможные проблемы
- **ModuleNotFoundError**: Убедитесь, что запускаете команды из корня проекта (`ITMO_MegaSchool`), а не из папки `src`.
- **API Error**: Проверьте правильность API ключа OpenAI и наличие средств на балансе.
